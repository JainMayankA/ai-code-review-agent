# Personal Review Rules

- Focus on real bugs, security risks, and meaningful maintainability issues.
- Do not comment on minor formatting or subjective style preferences.
- Always flag hardcoded secrets, tokens, passwords, and API keys.
- For Python code, check error handling, input validation, and type clarity.
- For API code, check authentication, authorization, and unsafe user input.
- Keep comments actionable and include a concrete suggested fix.
